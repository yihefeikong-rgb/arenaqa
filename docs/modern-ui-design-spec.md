# ArenaQA 现代化 UI 设计规范

本文档详细介绍了 ArenaQA 应用的现代化 UI 设计规范和实现细节。

## 设计理念

### 现代化设计原则
- **简约而不简单** — 减少视觉噪音，突出核心内容
- **功能导向** — 设计服务于功能，增强用户体验
- **一致性** — 统一的视觉语言贯穿整个应用
- **响应式** — 适配各种屏幕尺寸和设备

### 色彩系统

#### 主色调
- **主色**：#6366f1 (indigo-500) — 主要交互元素
- **辅助色**：#8b5cf6 (violet-500) — 渐变和强调
- **成功色**：#10b981 (emerald-500) — 成功状态
- **警告色**：#f59e0b (amber-500) — 警告状态
- **错误色**：#ef4444 (red-500) — 错误状态

#### 背景色系
- **背景**：#f8fafc (slate-50) — 页面背景
- **表面**：#ffffff — 卡片和面板背景
- **悬浮表面**：#f1f5f9 (slate-100) — 悬浮元素背景

#### 文字色系
- **主要文字**：#1e293b (slate-800) — 主要内容
- **次要文字**：#64748b (slate-500) — 辅助信息
- **禁用文字**：#94a3b8 (slate-400) — 不可用状态

### 字体系统

#### 字体家族
- **主要字体**：Geist — 现代化无衬线字体
- **等宽字体**：Geist_Mono — 代码和固定宽度文本

#### 字号层级
- **标题**：1.125rem - 1.75rem (18px - 28px)
- **正文**：0.875rem (14px)
- **小字**：0.75rem (12px)
- **标签**：0.6875rem (11px)

### 间距系统

#### 间距层级
- **xs**：0.25rem (4px)
- **sm**：0.5rem (8px)
- **md**：1rem (16px)
- **lg**：1.5rem (24px)
- **xl**：2rem (32px)

### 圆角系统

#### 圆角层级
- **sm**：0.375rem (6px) — 小组件
- **default**：0.5rem (8px) — 一般组件
- **lg**：0.75rem (12px) — 卡片
- **xl**：1rem (16px) — 大型面板
- **2xl**：1.5rem (24px) — 主要区域
- **3xl**：2rem (32px) — 全屏面板

### 阴影系统

#### 阴影层级
- **shadow-1**：轻微阴影，用于分隔
- **shadow-2**：标准阴影，用于卡片
- **shadow-3**：深度阴影，用于悬浮
- **shadow-4**：强化阴影，用于突出

## 组件设计

### 卡片组件
- **现代化卡片**：采用圆角、阴影和渐变边框
- **悬停效果**：轻微上移和阴影增强
- **焦点状态**：清晰的焦点指示器

### 按钮组件
- **渐变按钮**：使用现代化渐变背景
- **悬停动画**：平滑的变换和阴影效果
- **加载状态**：内置加载动画

### 输入组件
- **现代化输入框**：圆润边框和清晰焦点状态
- **标签系统**：清晰的标签和占位符
- **状态反馈**：成功、错误、警告状态

### 导航组件
- **玻璃态导航**：半透明背景和模糊效果
- **图标系统**：一致的图标风格
- **状态指示**：清晰的当前页面指示

## 动画效果

### 进入动画
- **淡入**：fadeIn — 从透明到不透明
- **上滑**：slideUp — 从下方滑入
- **缩放**：scaleIn — 从小到正常尺寸

### 交互动画
- **悬停脉冲**：hover-pulse — 轻微放大和阴影增强
- **浮动效果**：float — 轻微上下浮动
- **渐变流动**：gradient-move — 背景渐变流动

### 状态动画
- **加载旋转**：spinner — 旋转加载指示器
- **进度动画**：progressBar — 平滑的进度增长
- **折叠展开**：collapse — 平滑的高度变化

## 响应式设计

### 断点设置
- **移动端**：小于 768px
- **平板**：768px - 1024px
- **桌面端**：大于 1024px

### 布局适配
- **三栏布局**：支持拖拽调整，适应不同屏幕
- **内容优先**：移动设备上内容优先排列
- **触摸友好**：移动端增大点击区域

## 无障碍设计

### 键盘导航
- **焦点管理**：清晰的焦点指示器
- **Tab顺序**：符合逻辑的Tab顺序
- **快捷键**：提供常用功能快捷键

### 屏幕阅读器
- **ARIA标签**：提供必要的ARIA属性
- **语义化HTML**：使用正确的HTML元素
- **状态通知**：及时的状态变化通知

## 实现技术

### CSS变量系统
- **设计令牌**：通过CSS变量管理设计属性
- **主题切换**：支持明暗主题无缝切换
- **动态更新**：支持运行时动态更新

### 组件库
- **卡片系统**：现代卡片组件
- **动画系统**：预设动画类
- **布局系统**：响应式布局组件

### 样式管理
- **Tailwind CSS**：原子化CSS框架
- **CSS-in-JS**：组件级样式管理
- **主题系统**：主题配置和管理

## 使用指南

### 开发环境
1. 确保项目依赖已安装
2. 启动开发服务器
3. 应用现代化样式

### 组件使用
1. 导入所需组件
2. 使用预设的CSS类名
3. 遵循设计规范

### 定制化
1. 通过CSS变量自定义主题
2. 使用Tailwind配置文件扩展样式
3. 遵循设计系统保持一致性
