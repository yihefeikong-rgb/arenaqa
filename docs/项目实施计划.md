## Phase 1：核心 UI 替换

> **目标**：把旧的简陋界面全部换成 Kimi 原型的设计
> **原则**：不新增后端 API，只替换前端代码
> **施工顺序**：严格按 Kimi 建议的 1→10 顺序

### Step 1：安装依赖（1 条命令）

```bash
npm install react-markdown remark-gfm rehype-highlight rehype-katex
npm install react-syntax-highlighter
npm install @types/react-syntax-highlighter -D
```

| step-free-api | 跃问 | 多模态 |

| spark-free-api | 讯飞星火 | 办公场景 |
| deepseek-free-api | DeepSeek 网页 | GPT-4 平替 |

### 3.2 接入方式

所有 LLM-Red-Team 项目已是 **OpenAI 兼容 API**，可直接用现有的 `OpenAICompatProvider` 包装：

```typescript
// 示例：接入 kimi-free-api
taskManager.registerProvider('kimi-free',
  new OpenAICompatProvider({
    name: 'kimi-free',
    apiBase: 'http://localhost:8000/v1',  // Docker 本地部署
    apiKey: 'sk-dummy',
    modelId: 'kimi',
  })
);
```

## 关键技术决策（已确定）

| 决策 | 选择 | 理由 |
|------|------|------|
| Markdown 渲染 | react-markdown + rehype-highlight | Kimi 推荐，原型已验证 |
| 代码高亮 | react-syntax-highlighter + oneDark | 与原型一致 |
| 组件风格 | 全部手写（不对齐 shadcn/ui） | 对 Kimi 原型最少的偏差 |
| 状态管理 | Zustand（复用现有） | 项目已集成 |
| 布局方案 | CSS Grid（非 PanelGroup） | Kimi 原型用 Grid，更简单可控 |
| 深色模式 | CSS `data-theme` 属性切换 | Kimi 原型方案 |
